Cómo se aplicaron los patrones de diseño — DeliGo

Proyecto: Sistema de delivery online (Caso de estudio 4)
Stack: Frontend Angular · Backend Node.js · Base de datos SQLite
Repositorio: https://github.com/valerybrillyt/deligo

1. Introducción

En DeliGo se implementaron cuatro patrones de diseño del curso. Todos están en el backend (carpeta backend/patterns/). El frontend no implementa los patrones: consume el API y muestra el resultado (menú, tipos de entrega, extras, login). La base de datos solo guarda datos; los patrones son lógica en JavaScript.

3. Singleton

3.1 Conexión a la base de datos

Archivo: backend/patterns/singleton/databaseConnection.js

Garantiza una sola instancia de conexión a SQLite (database/deligo.db). Todas las rutas del API usan la misma conexión.

3.2 Servicio de autenticación

Archivo: backend/patterns/singleton/authService.js

Una sola instancia guarda las sesiones (tokens). Al iniciar sesión en la pantalla Login, el backend llama a authService.login().

4. Factory (Fábrica)

Archivo: backend/patterns/factory/orderFactory.js

Crea el tipo correcto de pedido según la opción del usuario:

TipoClaseTiempoEnvío
estándarPedidoEstandar45 min$3.50
expressPedidoExpress20 min$6.00
programadoPedidoProgramado60 min$2.00

Función: crearPedido(tipo, datos) — usada en Menú al confirmar pedido.

5. Decorator (Decorador)

Archivo: backend/patterns/decorator/orderDecorator.js

Añade extras al pedido sin modificar las clases de la Factory:

CódigoNombreCosto
propinaPropina para el repartidor$2.00
seguroProtección del pedido$1.50
ecoEmpaque sustentable$0.75

Función: aplicarDecoradores(pedido, extras)

6. Composite (Compuesto)

Archivo: backend/patterns/composite/menuComposite.js

Organiza el menú en árbol: categorías (compuestos) y productos (hojas).

Menú DeliGo (raíz)
 └── Pizzas → Pizza Margarita, Pizza Pepperoni
 └── Bebidas → Refresco 600ml

Función: construirMenuDesdeProductos(productos) — el menú se muestra agrupado por categoría.

7. Flujo al hacer un pedido

  1. Usuario abre Menú → Composite arma categorías y productos.
  2. Elige tipo de entrega → Factory crea el pedido (estándar, express o programado).
  3. Marca extras → Decorator suma propina, seguro o empaque eco.
  4. Confirma → Singleton de BD guarda en SQLite.
  5. Usuario ve el pedido en Seguimiento.

Ruta del API: POST /api/pedidos en backend/routes/api.js

8. Endpoints del API

RutaDescripción
GET /api/patronesDocumentación JSON de los 4 patrones
GET /api/pedidos/tiposTipos de la Factory
GET /api/pedidos/extrasExtras del Decorator
GET /api/restaurantes/:id/menuMenú con árbol Composite
POST /api/usuarios/loginLogin (Singleton authService)
POST /api/pedidosCrear pedido (Factory + Decorator)

Pantalla académica: http://localhost:4200/patrones

9. Frases para la exposición

  • Singleton: Una sola conexión a la BD y un solo servicio de sesiones.
  • Factory: La fábrica crea el pedido según el tipo de entrega elegido.
  • Decorator: Los extras se agregan envolviendo el pedido sin cambiar la fábrica.
  • Composite: El menú se organiza por categorías en forma de árbol.
